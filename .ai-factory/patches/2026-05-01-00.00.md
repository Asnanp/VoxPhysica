# Stabilize 3cm height MAE training path

**Date:** 2026-05-01 00:00
**Files:** src/training/trainer.py, src/models/vocalmorph_v2/config.py, src/models/vocalmorph_v2/model.py, src/models/vocalmorph_v2/losses.py, configs/pibnn_rtx3060_3cm_power.yaml, START_3CM_POWER_TRAINING.bat, tests
**Severity:** high

## Problem

The existing elite training profiles targeted low height MAE, but strict audit outputs were still around 4.2 cm validation speaker MAE and 5.9 cm test speaker MAE. The 15-epoch profile was especially aggressive and could not reliably consolidate gains.

## Root Cause

Three training mechanics were fighting the target. OneCycleLR was configured as if `total_steps` meant epochs and was stepped once per epoch instead of once per optimizer update. V2 mixup accepted `mixup_p` in config but effectively applied mixup to every eligible sample because the model had no probability gate. The focal height loss used an inverted residual weight, which could suppress hard height examples such as short/tall slices.

## Solution

OneCycleLR now derives optimizer-update steps from the dataloader and steps after optimizer updates. V2 exposes `mixup_probability`, maps it from `training.augmentation.mixup_p`, and gates mixup accordingly. Height focal weighting now emphasizes larger residuals and normalizes the weights. SWA winners are saved back to `best.ckpt`, and a dedicated `pibnn_rtx3060_3cm_power.yaml` config provides a stable height-focused launch profile.

## Prevention

Add tests around scheduler step cadence, mixup probability mapping, and hard-example height focal behavior. Treat documented training knobs as contract: if a YAML key exists, verify code consumes it.
